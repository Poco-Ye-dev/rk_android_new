## 概述

本文档描述了将爱科微(AIC)的AIC8800D80 USB WiFi/BT Combo模块移植到Rockchip RK3576 Android 14平台的具体步骤。该模块支持WiFi 802.11a/b/g/n/ac/ax和Bluetooth 5.3协议。

**关键特性**：

- USB接口通信，设备可自动枚举，无需额外的DTS配置
- 蓝牙USB驱动节点：`/dev/aicbt_dev`
- UART设备节点：`/dev/ttyS1` 或 `/dev/ttyS2`（实际硬接调整）

***

## 硬件要求与接口配置

### USB接口配置

AIC8800D80 USB模块采用标准USB 2.0/3.0接口，RK3576平台无需特殊的DTS配置即可支持设备枚举。USB控制器会自动识别设备ID：

```
Vendor ID: 0xa69c
Product ID: 0x8d80
```

当USB模块插入后，系统会在`dmesg`中看到类似以下日志：

```
usb 1-1: new high-speed USB device number 2 using dwc3
aic_btusb 1-1:1.0: AIC BTUSB device attached
aic_btusb: BTUSB uploaded to /dev/aic_btusb
```

### 蓝牙设备节点权限

在`init.connectivity.rc`中配置蓝牙设备节点权限：

```bash
# for aic bt usb
chmod 0660 /dev/aic_btusb
chown bluetooth system /dev/aic_btusb
chmod 0660 /dev/aicbt_dev
chown bluetooth system /dev/aicbt_dev
```

这些权限设置确保蓝牙服务进程能够正常访问设备节点。

## 驱动文件结构

### 内核驱动位置

USB蓝牙驱动位于SDK目录：

```
USB/driver_fw/drivers/aic_btusb/
```

主要驱动文件：

- `aic_btusb.c` - USB蓝牙主驱动

USB WiFi驱动位于：

```
USB/driver_fw/drivers/aic8800/
```

WiFi驱动包含两个主要模块：

- `aic_load_fw/` - 固件加载模块
- `aic8800_fdrv/` - WiFi主驱动

### 驱动模块加载顺序

根据`frameworks_opt_diff.patch`中的修改，驱动加载顺序如下：

```cpp
// 1. 首先加载固件加载驱动
insmod(AIC8800_BSP_DRIVER_MODULE_PATH, "");  // /vendor/lib/modules/aic_load_fw.ko

// 2. 等待200ms确保固件加载完成
usleep(200000);

// 3. 然后加载WiFi主驱动
insmod(AIC8800_DRIVER_MODULE_NAME_PATH, "");  // /vendor/lib/modules/aic8800_fdrv.ko
```

### 内核模块路径配置

在`wifi_hal_common.cpp`中定义驱动路径：

```cpp
#define WIFI_MODULE_PATH        "/vendor/lib/modules/"
#define AIC8800_BSP_DRIVER_MODULE_PATH   WIFI_MODULE_PATH"aic_load_fw.ko"
#define AIC8800_DRIVER_MODULE_NAME_PATH WIFI_MODULE_PATH"aic8800_fdrv.ko"
```

## HAL层配置

### 目录结构

WiFi HAL和蓝牙Vendor HAL统一放置在`hardware/aic/`目录下：

```
hardware/aic/
├── aicbt/                    # 蓝牙Vendor HAL
│   ├── Android.mk
│   ├── aicbt.mk
│   ├── libbt/
│   │   ├── Android.mk
│   │   ├── src/
│   │   │   ├── hardware.c
│   │   │   ├── hardware_uart.c    # UART通信实现
│   │   │   ├── hardware_usb.c      # USB通信实现
│   │   │   ├── bt_vendor_aic.c
│   │   │   └── ...
│   │   └── include/
│   └── vendor/
│       └── etc/bluetooth/aicbt.conf
├── wlan/                     # WiFi HAL
│   ├── Android.mk
│   ├── wifi_hal/
│   └── config/
└── firmware/                  # 固件文件
    └── USB/aic8800D80N/ 或 aic8800D80X2/
```

### 蓝牙HAL配置

在`hardware/aic/aicbt/aicbt.mk`中配置：

```makefile
BOARD_HAVE_BLUETOOTH := true

PRODUCT_COPY_FILES += \
    $(CUR_PATH)/vendor/etc/bluetooth/aicbt.conf:vendor/etc/bluetooth/aicbt.conf

PRODUCT_PACKAGES += \
    libbt-vendor-aic
```

### 编译产物

编译完成后生成以下库文件：

| 库文件                   | 说明            |
| --------------------- | ------------- |
| `libbt-vendor-aic.so` | 蓝牙Vendor HAL库 |
| `libwifi-hal-aic.a`   | WiFi HAL静态库   |

这些库文件会被打包到system分区，系统启动时自动加载。

## 固件配置

### 固件目录结构

WiFi和蓝牙固件放置在`hardware/aic/firmware/`目录下：

```
hardware/aic/firmware/
├── USB/
│   ├── aic8800D80N/           # AIC8800D80N固件
│   │   ├── fmacfw_btusb_8800d80n_u02.bin
│   │   ├── fmacfw_calib_8800d80n_u02.bin
│   │   ├── fmacfw_patch_8800d80n_u02.bin
│   │   └── lmacfw_rf_8800d80n.bin
│   └── aic8800D80X2/          # AIC8800D80X2固件
│       ├── fmacfw_8800d80x2.bin
│       ├── aicbt.conf
│       └── ...
└── device-aic.mk               # 固件打包配置
```

### 固件打包配置

在`device-aic.mk`中配置固件打包规则：

```makefile
AICS_SUBDIRS := $(wildcard hardware/aic/firmware/USB/*/)
AICS_DEST_DIRS := $(TARGET_COPY_OUT_VENDOR)/etc/firmware

PRODUCT_COPY_FILES += \
    $(foreach DIR, $(AICS_SUBDIRS), \
        $(foreach file, $(notdir $(shell ls $(DIR)/*.bin $(DIR)/*.txt)), \
            $(DIR)/$(file):$(AICS_DEST_DIRS)/$(file) \
        ) \
    )
```

此配置将固件文件复制到系统镜像的`/vendor/etc/firmware/`目录。

## 编译与部署

### 编译步骤

1. **配置编译选项**

在产品配置文件中启用AIC蓝牙：

```bash
BOARD_HAVE_BLUETOOTH_AIC := true
```

1. **编译内核驱动**

```bash
# 编译USB蓝牙驱动
cd USB/driver_fw/drivers/aic_btusb
make

# 编译WiFi驱动
cd USB/driver_fw/drivers/aic8800
make
```

1. **编译Android HAL**

```bash
# 编译蓝牙Vendor HAL
m libbt-vendor-aic

# 编译WiFi HAL
m libwifi-hal-aic
```

1. **完整系统编译**

```bash
make -j$(nproc)
```

### 部署文件清单

| 文件类型  | 目标路径                                  |
| ----- | ------------------------------------- |
| 内核模块  | `/vendor/lib/modules/aic_btusb.ko`    |
| 内核模块  | `/vendor/lib/modules/aic_load_fw.ko`  |
| 内核模块  | `/vendor/lib/modules/aic8800_fdrv.ko` |
| 蓝牙库   | `/vendor/lib/libbt-vendor-aic.so`     |
| WiFi库 | `/vendor/lib/libwifi-hal-aic.a`       |
| 固件    | `/vendor/etc/firmware/*.bin`          |
| 配置文件  | `/vendor/etc/bluetooth/aicbt.conf`    |

### 常见问题排查

| 问题现象      | 可能原因        | 解决方案                 |
| --------- | ----------- | -------------------- |
| USB设备不枚举  | 内核未编译USB驱动  | 检查内核配置，启用CONFIG\_USB |
| 蓝牙服务启动失败  | SELinux权限不足 | 检查并修正sepolicy配置      |
| WiFi扫描无结果 | 固件未加载       | 检查固件路径和加载顺序          |
| 设备节点权限错误  | ueventd配置缺失 | 添加设备节点权限配置           |

## 附录C：硬件参考设计说明

### AIC8800D80 EVB V5硬件参数

根据爱科微提供的AIC8800D80 EVB V5参考设计，主要硬件参数如下：

**电源要求**：

- 工作电压：3.3V ± 5%
- 工作电流：典型值350mA，峰值600mA
- 建议使用LDO供电，输出电流不小于600mA

**USB接口**：

- 标准USB 2.0 High Speed接口
- 支持USB 3.0向下兼容
- 建议使用带ESD保护的USB座

**射频参数**：

- WiFi频段：2.4GHz (2400-2483.5MHz)，5GHz (5150-5850MHz)
- 蓝牙频段：2402-2480MHz
- 天线接口：ipex4座或邮票孔
- 50Ω阻抗匹配

**PCB布局建议**：

- USB差分线90Ω阻抗控制
- 射频走线50Ω阻抗控制
- 电源滤波电容靠近芯片管脚
- 建议4层板设计

### AIC8800D80P EVB V2硬件参数

**电源管理**：

- 支持USB供电模式
- 支持外接LDO供电
- 支持低功耗模式

**时钟要求**：

- 集成26MHz晶体振荡器
- 精度要求：±25ppm

**接口定义**：

| 引脚          | 功能        | 说明      |
| ----------- | --------- | ------- |
| USB\_D+     | USB Data+ | 连接USB主机 |
| USB\_D-     | USB Data- | 连接USB主机 |
| GND         | 接地        | 电源地     |
| VBUS        | USB电源     | 3.3V输入  |
| WL\_BT\_ANT | WiFi/蓝牙天线 | 射频输出    |
